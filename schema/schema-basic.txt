In PostgreSQL, a schema is a logical namespace inside a database that organizes objects like tables, views, functions, and types.

Think of it as:

a folder inside a database




Postgres cluster
 └── database
      └── schema
           ├── table
           ├── view
           ├── function
           └── index



✅
Default schemas

| Schema               | Purpose                         |
| -------------------- | ------------------------------- |
| `public`             | Default user schema             |
| `pg_catalog`         | System tables & functions       |
| `information_schema` | SQL-standard metadata           |
| `pg_toast`           | Large object storage (internal) |




You normally don’t touch system schemas.



Fully qualified names

schema_name.object_name


Example:

sales.orders



✅
search_path (super important)

Postgres resolves object names using search_path.
